── HOW THE R2 PIPELINE WORKS (so you can trust/repeat it) ──
gen-<city>-r2-workflow.mjs reads the existing data/<city>.json, bakes an EXCLUDE
list (existing place names) + fresh discovery angles into scripts/<city>-r2-
workflow.js (a Workflow-tool script: loop-until-dry discovery excluding existing,
then verify every fresh candidate). Run it with the Workflow tool; it returns
{kept,dropped,counts}. Save result.kept to data/_<city>_r2.json. build-<city>-
merge.mjs APPENDS those to the existing city json (seeds dedup state from current
places, skips dups/out-of-scope/bad-coord), then dedup-city.mjs is the same-shop
safety net. NOTE: a workflow can be resumed from its runId (cached agents return
instantly) — but if it half-finished, its kept[] in the .output already holds the
verified places; just extract & merge rather than re-running 500+ agents.
