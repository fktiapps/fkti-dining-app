STRUCTURE
- index.html ........ the whole app (HTML/CSS/JS). Manifest-driven city registry + Leaflet map.
- sw.js ............. service worker (bump VERSION on shell changes).
- wrangler.toml ..... Pages config (name, SUMMARIES KV). Secrets via `npm run secret`, not here.
- functions/api/ .... Cloudflare Functions: menu-scan.js + label-scan.js (Mistral Pixtral vision, now
                      wired); nearby.js (live OSM proxy for "food near me"); chef-bio*, place-summary, health.
- data/manifest.json  city registry {version,default_city,cities:[{id,name,file,center,bounds,free}]}.
- data/<city>.json .. per-city {city,subtitle,center,bounds,tz,places:[...]}. places carry gf_confidence
                      (dedicated|high|options|ask|no), vegan_status (full|options|limited|ask|no), category
                      (BOTH|GF|VEGAN|SHOJIN|OMNI|MOM_AND_POP), cuisine_type, hours, bio, safety, dcp, etc.
- data/<city>_konbini.json / _grocery.json ... survival layers (OSM points + chain-keyed GF/vegan KB).
- data/menupedia.json .. {hundred:[100 dishes], foundations:[16], lore:{key:{history,name,culture,taste,gf,vegan}}}.
- data/stations.json ... 🚉 station survival guide (6 stations) — schema {name,intro,sections:[{icon,h,
                      verdict,where,rows:[{t,gf,vegan,speed,note}],tip}],playbook:{tight,gf,vegan}}.
- data/_<city>_disc.json ... raw verified sweep output (input to build-spot.mjs).
- scripts/ ......... city-sweep tooling: spot-configs.mjs (per-city radius/config, incl. himeji),
                      gen-spot-workflow.mjs (generates the workflow), spot-<city>-workflow.js (Workflow-tool
                      script: Haiku discover → Sonnet verify), build-spot.mjs (<city>_disc.json → <city>.json
                      + registers in manifest + bumps SW). Also build-dcp.mjs, ramen enrich, menu pass, dedup.
- MOBILE_OPS.md ... git/Pages mobile playbook.  FOOD_NEAR_ME.md ... "food near me" feature spec.

CITIES (9): kyoto(default), kanazawa, nara, hiroshima, nagano, toba(&Ise), nagoya, HIMEJI (added
